# UI Review Skill

**Trigger Condition**: New screen or component built.

## Input
- Figma link or component code

## Execution Steps
1. Review the UI component against the SpendOS Design System palette and typography.
2. Conduct an accessibility audit (color contrast, ARIA labels).
3. Identify UX friction points.
4. Check for consistency with existing components.
5. Provide a WCAG rating.

## Output
Accessibility audit report, UX friction analysis, consistency issues list, and WCAG rating.
