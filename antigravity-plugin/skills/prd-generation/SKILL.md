# PRD Generation Skill

**Trigger Condition**: New product feature requested.

## Input
- Feature description
- User need
- Business goal

## Execution Steps
1. Analyze the feature description, user need, and business goal.
2. Draft user stories in the format: As a [user], I want to [action] so that [benefit].
3. Define acceptance criteria.
4. Identify potential risks.
5. Combine into a structured PRD section.

## Output
Structured PRD section including stories, acceptance criteria, and risks.
